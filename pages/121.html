Microcontroller System Design - ECE 121

This class had me programming an UNO32 microcontroller in the C langauge with an emphasis on low-level direct hardware manipulation. This meant creating my own APIs to set control registers on the CPU to interface with various external systems. Some of the tools utilized include timers, Input/Output Compare, ADC, PWM, interrupts, bus and memory organization, DMA, SPI, I2C, device driver programming, serial packet communication, and clocking.

The following is one lab I did which utilized I2C to communicate with the EEPROM which was on the board. An API was created to write bytes and pages from specified memory addresses as well as read them back. This information was then displayed on a computer terminal over UART. This lab also required reading from the ADC CPU module and applying a basic filter to the readings which was then saved to the EEPROM. The code for this lab can be found here .

This lab involved writing an API to utilize the Output Capture CPU module to output an adjustable PWM signal. This PWM signal was then used to control the speed of a DC motor. The speed was fedback into the microcontroller using a rotary encoder and this data was used to create a control system to change the PWM signal to command motor speeds. The SPI API used to communicate with the rotary encoder was written as part of a previous lab. The code for this lab can be found here .
