Senior Design Project: Tactile Stimulation - ECE 129

I was one member of a two person team in my Senior Design Project. Our project aimed to simulate virtual reality textures using transcutaneous electrical nerve stimulation. Creating the electrical nerve interface required circuit design and modeling, iterative prototyping, additive manufacturing, and microcontroller programming to implement specific waveforms. The Senior Design Project was a three-part course so a lot of time was allocated for each deliverable. The hardwork has been recognized and our team received a prize in the 2022 Santa Cruz Launchpad Pitch Competition.

The following button can be pressed to show my team's design brief from the first few months of the course. Pressing the button again will collapse the PDF. This document served as an amalgamation of all our initial research. In addition to providing context and background for our idea, it also highlighted some potential approaches to solving the problem. This document also served to set the groundwork for our work flow moving forward through the year.

This class served as my first foray into using full team management tools. This included a Gantt chart for tracking task progression as well as discrete Sprints each capped with a report and design brief. Design briefs were in the form of a 2-3 hour presentation with the teaching staff where we discussed the progress of the Sprint.

The following is an example of a Sprint Report. This course required the team to fill out a report at the end of each Sprint in order to detail the progress made. This also served to create a framework for starting the following Sprint. Sprint reports were created after design briefs so as to include instructor feedback from the Sprint.

The following button includes an example of notes taken for a specific task. This task was the initial creation of the code to run on our Microcontroller. This Microcontroller was responsible for timing the signals which we will ultimately driving through the surface electrodes. I created these notes for internal team use and they were not required for a grade or submitted. It was created as a reference to help with future development of the project by aiding in the understanding of the code. The corresponding code as well as the final code can be found here  in my GitHub.

A similar write-up of a task was created to document the process of developing the electrode driving circuit. This does not include the final design as many revisions were necessary. This write-up did however help my partner understand the work I had done which allowed him to better help me continue development.

The Final Report served to fully detail the project as it was at the end of the course. It was created to describe the device and not the steps leading up to each design decision. This was done keep the document as concise and as relevant as possible.
